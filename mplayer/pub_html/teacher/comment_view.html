인공지능과 수학(선형회귀분석 편)

덕영고등학교 한창수 수학교사

 ‘인공지능’이란 인간만이 할 수 있다고 여겨지는 기능과 작업을 기계가 수행하는 것을 의미합니다. 이러한 인공지능을 컴퓨터에서 구현하려면 무엇이 필요할까요? 간단하게 이야기하면 ‘컴퓨터를 어떻게 공부시킬 수 있을까?’ 로 표현할 수 있습니다. 그럼 컴퓨터를 공부시키기 위해서는 무엇이 필요할까요? 아마 사람이 공부할 때처럼 책, 공부 방법, 그리고 노력이 필요할 것입니다. 그럼 이것을 어떻게 컴퓨터에게 줄 수 있을까요? 아마 책은 데이터로, 공부 방법은 알고리즘으로, 노력은 학습 횟수로 프로그래밍 될 것입니다.(이것을 기계학습이라고도 합니다.)

 여기서 우리는 알고리즘에 주목해야 합니다. 알고리즘은 문제를 해결하기 위한 절차나 방법을 의미하는데요. 이는 인공지능을 구현하는데 있어 ‘어떻게 구현할 것인가?’에 대한 방향을 설정해주기 때문입니다.

 그럼 오늘은 인공지능의 알고리즘 중에서 가장 간단한 <선형 회귀 분석>에 대해서 알아보도록 하겠습니다. 선형 회귀 분석은 한 개 이상의 독립변수와 종속변수를 선형 함수 관계로 예측하는 대표적인 분석 알고리즘입니다.

 예를 들어 석 달 동안 식사량, 식사시간, 운동량, 운동시간, 몸무게를 매일 매일 기록한 데이터를 준비해 선형 회귀 분석으로 미래의 몸무게를 예측하는 인공지능 시스템을 만든다고 생각해봅시다. 그러면 식사량, 식사시간, 운동량, 운동시간은 독립변수이고, 몸무게가 종속변수가 되어 선형 함수 관계로 예측하게 되는 것입니다.

 그럼 컴퓨터는 어떻게 선형 회귀 분석을 하게 될까요? (이것을 인공지능에서는 학습된 모델을 만든다고 표현하기도 합니다.) 간단하게 설명하면 선형 함수식 h = Wx + b (h는 예측값, hypothesis를 의미)에서 독립변수의 데이터 x값과 랜덤하게 생성되는 W값과 b값으로 h값을 계산하게 됩니다. 이렇게 계산된 h값을 데이터의 종속변수인 y값의 차이를 비교합니다. 이 과정을 반복하면서 h값과 y값의 차이가 적은 W값과 b값을 찾게 됩니다. 그러면 데이터를 잘 반영한 선형 함수식 h = Wx + b가 만들어지고, 사용자가 임의의 데이터(독립변수)를 입력하면 만들어진 선형 함수식으로 예측을 하게 됩니다.

 여기서 우리는 한 가지 궁금증을 가지게 됩니다. 어떻게 예측된 h값과 데이터의 종속변수인 y값의 차이를 비교할까요? 바로 비용 함수(cost)라고 불리는 방법을 이용합니다.

* 비용 함수(cost)
  선형 회귀 분석에서 cost는 h값과 y값의 차를 제곱하여 평균을 계산한 값을 의미합니다.
  통계에서는 이를 분산이라고 하고, 함수식으로 표현하면 아래와 같습니다.

(n은 데이터값의 개수)

  좀 더 자세히 생각해봅시다. (독립변수가 하나라고 생각하고) x축을 독립변수, y축을 종속변수로 하는 좌표평면에 데이터를 점으로 표현하고, 기계학습으로 생성된 선형 함수식을 직선으로 나타내 봅시다.

  이 직선은 데이터를 잘 나타내는 직선일까요? 그렇게 보이기는 하지만 그렇다고 확신할 수 없습니다. 그래서 아래와 같이 (hypothesis – y) 제곱의 평균인 cost를 계산해 데이터를 잘 나타내는 직선을 찾으려고 하는 것입니다.

  아! 그렇군요.. 그런데 또 궁금한 게 있습니다. cost를 계산해 데이터를 잘 나타내는 직선을 찾는 건 어떻게 하는 걸까요? 바로 최적화 함수(optimizer)라고 불리우는 방법을 이용합니다.

* 최적화 함수(optimizer)
  optimizer는 cost를 이용해 데이터를 가장 잘 나타내는 함수식을 찾는 방법을 의미합니다. 인공지능에서는 최적의 모델의 찾는 방법이라고 생각하면 됩니다.
  선형 회귀 분석에서 optimizer는 경사하강법(gradient descent)을 주로 사용합니다.

 이건 또 무슨 뜻일까요? 좀 어렵지만 최대한 쉽게 설명해보겠습니다. 먼저 경사하강법(gradient descent)은 용어 그대로 내려가면서 찾겠는다는 의미입니다. 즉, 내려가면서 cost를 찾는데, cost가 가장 작은 값을 찾겠는다는 의미입니다. 그럼 뭐가 내려간다는 걸까요?
그건 미분계수가 0에 가까워지는 cost를 찾기 위해 내려간다는 의미입니다. 갑자기 미분계수가 등장해서 어렵게 느껴질 수 도 있습니다. 조금만 더 노력해봅시다.

  cost를 함수식으로 표현하면 아래와 같다는 것 기억하시죠?

(n은 데이터값의 개수)

  cost는 이차함수이고, 그래프 모양은 포물선입니다. 그러면 x축을 hypothesis, y축을 cost로 하는 좌표평면에 포물선을 그리면 cost 함수의 그래프라고 생각할 수 있습니다.

  그럼 cost의 가장 작은 값은 포물선의 꼭지점이 되겠죠? 즉, 포물선의 꼭지점에 가까이 가는 hypothesis값을 찾아 cost를 구하면 됩니다.
이렇게 포물선의 꼭지점에 가까이 가는 hypothesis값을 찾을 때 미분계수가 사용됩니다. 미분계수는 특정한 점을 접점으로 하는 접선의 기울기를 의미합니다. 정리하면 포물선의 꼭지점을 접점으로 하는 접선의 기울기는 0이므로 cost가 가장 작은 값을 찾기 위해 미분계수가 0에 가까워지는 cost를 찾는 것입니다.

  이렇게 최적화 함수를 통해 cost를 찾으면 준비된 데이터를 가장 잘 나타내는 직선 W값과 b값을 찾게 되고, 선형 회귀 함수식 hypothesis = Wx + b을 만들게 됩니다. 이 후, 사용자가 임의의 데이터를 입력하면 만들어진 선형 회귀 함수식으로 예측을 하게 됩니다. 이것이 선형 회귀 분석으로 인공지능 시스템을 만들 때 학습된 모델을 만들고, 예측하는 방법입니다.
